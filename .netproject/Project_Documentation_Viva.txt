============================================================
PROJECT EXPLANATION FOR VIVA: .NET E-COMMERCE SYSTEM
============================================================

1. WHY I CHOSE THIS PROJECT
------------------------------------------------------------
I chose this project because it allows me to combine all the academic experiments into one cohesive, real-world application instead of writing scattered, disconnected programs. This E-Commerce system perfectly demonstrates the full stack of modern .NET development. It seamlessly integrates basic C#, an ASP.NET Core MVC web application, and a modern RESTful Web API, showing that I know how different parts of a software architecture communicate in a professional setup.

2. TOOLS & TECHNOLOGIES USED
------------------------------------------------------------
* Backend & Core Logic: C# and .NET 8 (or current active version)
* Web Frameworks: ASP.NET Core MVC (for main UI) and ASP.NET Core Web API (for fetching data)
* Database & ORM: Entity Framework (EF) Core (Code-First approach) and SQLite (ecommerce.db)
* Frontend: HTML, CSS, JavaScript, and Razor Pages (.cshtml files)
* IDE: Visual Studio / VS Code

3. REFERENCE TO ACADEMIC EXPERIMENTS (Exps)
------------------------------------------------------------
During the viva, you can explain that this project implements your lab experiments as follows:
* Exp (C# Basics & OOP): Demonstrated through the C# classes, objects, and ConsoleApp.
* Exp (ASP.NET Core & MVC Arch): The foundation of the "MyApp.MVCApp" structure.
* Exp (EF Core & CRUD Operations): Demonstrated mainly in the "CustomersController", where we Create, Read, Update, and Delete data in the database.
* Exp (State Management): Handled in "CartController" to remember what the user put in their cart.
* Exp (Authentication): Done in "AuthController" for user registration and login.
* Exp (Creating & Consuming Web API): I created "MyApp.ProductAPI" to supply the product list, and the MVC App consumes it.

4. PROJECT WORKFLOW (The Flow of Data)
------------------------------------------------------------
1. Starting the Apps: Both the API and the MVC app must be running. 
2. Landing Page: The user opens the MVC App in a browser. The HomeController asks the Product API for the list of available products, which it displays on the screen.
3. Registration/Login: If a user wants to buy something, they register. The AuthController saves their details securely into the SQLite database.
4. Managing Customers: An admin can use the pages generated by the CustomersController to view, add, or edit customer details (This is pure database CRUD in action).
5. Shopping Cart: As the user clicks "Add to Cart", the CartController saves these items into the application's memory (Session), so they don't lose their cart when clicking around.

5. FILE-BY-FILE EXPLANATION (Explain what each file does)
------------------------------------------------------------
It is helpful to explain the project in 3 main parts: ConsoleApp, MVCApp, and ProductAPI.

[ PART A: MyApp.ConsoleApp ]
* Program.cs: This is used for basic, raw logic testing. It proves the background concepts work without the complexity of a web browser.

[ PART B: MyApp.ProductAPI (The Backend Data Supplier) ]
* Models/Product.cs: The blueprint for what a "Product" is (its ID, Name, Price).
* Controllers/ProductController.cs: The API endpoints (URLs). When requested, it sends out product data in "JSON" format to the frontend.
* Program.cs: Configures the API server to listen to requests.

[ PART C: MyApp.MVCApp (The Frontend & Main Application) ]

-> THE MODELS (Data Structures)
* Models/Customer.cs: Represents the user (ID, Email, Password).
* Models/CartItem.cs: Represents a product added to the cart along with its chosen quantity.

-> THE DATA TIER (Database)
* ecommerce.db: Our actual SQLite database file storing the records permanently.
* Data/AppDbContext.cs: The Entity Framework Core "bridge" file. It talks to the database for us so we do not have to write raw SQL statements.

-> THE CONTROLLERS (The Brains / Logic)
* Controllers/HomeController.cs: Handles loading the main website pages and fetches products from the Product API.
* Controllers/CustomersController.cs: Contains the methods to Add, View, Edit, and Delete Customers (CRUD Operations).
* Controllers/AuthController.cs: Handles verifying login credentials against the database.
* Controllers/CartController.cs: Contains logic to add items to the cart and view the cart.

-> THE VIEWS (The UI Screens)
* Views/Home/, Views/Customers/, Views/Cart/, etc.: These folders contain `.cshtml` files (HTML mixed with C# code). They are the actual screens the user sees and interacts with.

-> THE CONFIGURATION
* Program.cs: The central engine configuration for the MVC App. It configures the database connection, session memory, and web page routing.
* appsettings.json: Stores key settings like connection strings to the database and API URLs.
